### ‚úèÔ∏è Mutation Hooks (Escritura de Datos)

#### `useCreateGuest()`


- ‚úÖ **Optimistic Update**: UI se actualiza inmediatamente

## üóÑÔ∏è Zustand Store (UI State)

## ‚ú® Caracter√≠sticas Principales

### üöÄ Optimistic Updates

Todas las mutations (create, update, delete, bulk) implementan **optimistic updates**:

1. **UI se actualiza inmediatamente** (sin esperar respuesta del servidor)
2. **Mejor UX**: El usuario ve los cambios al instante
3. **Rollback autom√°tico** si el servidor retorna error
4. **Toast notifications** para feedback visual

### üîÑ Cache Invalidation

El cache se invalida autom√°ticamente cuando es necesario:

- Crear invitado ‚Üí Invalida listas y estad√≠sticas
- Actualizar invitado ‚Üí Invalida detalle, listas y estad√≠sticas
- Eliminar invitado ‚Üí Invalida listas y estad√≠sticas
- Bulk operations ‚Üí Invalida listas y estad√≠sticas

### üíæ Persistencia

El Zustand store persiste autom√°ticamente en LocalStorage:
- ‚úÖ View mode (table/grid)
- ‚úÖ Page size
- ‚úÖ Sort preferences

Los filtros y selecci√≥n **NO** se persisten (reset al recargar p√°gina).
